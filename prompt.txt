Implementa customerProfiles con código automático de cliente para FoodSPV 2.0.

Objetivo:
Separar clientes del sistema interno de usuarios.
No crear rol customer.
Los clientes no hacen login.
El cliente queda registrado automáticamente en su primer pedido mediante un código de cliente.

Stack:
- Next.js App Router
- Firestore
- TypeScript
- Arquitectura modular en src/modules

Reglas:
- No tocar roles internos salvo si existe customer, eliminarlo.
- No crear login para clientes.
- No usar any.
- No romper checkout actual.
- No tocar webhook ni WhatsApp salvo el texto de comanda si ya incluye cliente.
- Mantener compatibilidad con pedidos existentes.

Implementar:

3. Crear servicio server:

src/modules/customers/server/upsertCustomerProfile.ts

Debe recibir:
- tenantId
- tenantSlug
- displayName
- phone
- customerCode opcional
- address opcional

Lógica:
A) Si customerCode existe:
   - buscar tenants/{tenantId}/customerCodes/{customerCode}
   - si existe, obtener customerId
   - actualizar customerProfile:
     - displayName si cambió
     - phone si cambió
     - lastOrderAt
     - totalOrders + 1
     - agregar address si viene y no es duplicada simple
   - devolver customerProfile

B) Si customerCode no existe:
   - generar customerCode único
   - crear customerProfile en tenants/{tenantId}/customerProfiles/{customerId}
   - crear índice tenants/{tenantId}/customerCodes/{customerCode}
   - totalOrders = 1
   - blocked = false
   - devolver customerProfile

C) Si customerCode existe pero no se encuentra:
   - no bloquear pedido
   - crear nuevo perfil o devolver error controlado según implementación actual
   - recomendado: crear nuevo perfil y regresar warning: "Código no encontrado, se generó uno nuevo."
